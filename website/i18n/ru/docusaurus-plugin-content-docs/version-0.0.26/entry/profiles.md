# Вкладка Profiles

Вкладка Profiles управляет изолированными профилями глобальной памяти. Профили предотвращают утечку памяти клиентов, компаний и личной памяти за пределы соответствующих областей.

## Имя профиля (Profile name)

Именованный контекст памяти, например `personal`, `client-a`, `team-platform`. Используйте буквы, цифры, `.`, `_`, `-`; избегайте пробелов и конфиденциальных имен. Имена должны соответствовать шаблону `^[a-zA-Z0-9][a-zA-Z0-9._-]{0,63}$`.

## Глобальный путь (Global path)

Папка файловой системы, в которой хранится профиль. Рекомендуется использовать абсолютные пути вне временных папок; убедитесь в наличии прав на запись.

## Активировать (Activate)

Делает профиль активным для разрешения по умолчанию на уровне пользователя. Переключение с личной памяти на рабочую/память клиента влияет на будущие загрузки и сохранения.

<RiskCallout level="caution">
Активация профиля меняет глобальную память, которая будет использоваться при будущих загрузках и сохранениях. Подтвердите имя профиля перед активацией.
</RiskCallout>

## Удалить (Delete)

Удаляет регистрацию профиля. Метаданные профиля удаляются; файлы памяти могут по-прежнему существовать на диске, если только поведение кода не изменится. Проверьте папку перед использованием удаления.

## Эквивалент в CLI

```bash
engram profile create personal --global-path ~/Documents/engram-personal --use
engram profile use company --workspace
engram profile merge personal company --dry-run
```
